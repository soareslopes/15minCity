# Outputs

This directory contains all data and visualizations automatically generated by the **Urban Accessibility** pipeline. The structure is organized to separate consolidated results, city-specific geospatial data, intermediate processing files (cache), and comparative analyses.

## Content Structure

### 1. Tabular Results (`/output/`)
* **`results_final.csv`**: Consolidated table containing final accessibility indicators and inequality metrics for all processed cities.
* **`results_partial.csv`**: Backup of results during processing (useful for long-running executions).

### 2. Geospatial Data (`/output/gpkg/`)
Contains **GeoPackage** files with the final hexagonal grid (H3) for each city. These files are ideal for GIS analysis (QGIS/ArcGIS).
* *Included layers:* Hexagonal geometry, destination counts per category, travel times (Dijkstra/Pandana), and population data (GHSL).

### 3. City-Specific Repository (`/output/cities/[CITY_ID]/`)
Individual folder for each processed city, containing:
* **`network.graphml`**: Pedestrian network graph extracted from OSM.
* **`[ID].osm.pbf`**: Local OpenStreetMap extract used in the analysis.
* **`boundary.gpkg`**: Polygon of the city's administrative/urban boundary.
* **`map_total_dest_30min.png`**: Static map for a quick visualization of accessibility.
* **`ghsl/`**: Local clips of population density data.

### 4. Comparative Analyses (`/output/figures/`)
Charts generated in Step 9 (when 2 or more cities are processed) to enable urban benchmarking:
* **`density_vs_accessibility.png`**: Correlation between urban density and access times.
* **`inequity_distributions.png`**: Statistical distribution of inequalities.
* **`gini_vs_density.png`**: Analysis of the Gini Index against population density.

### 5. Data Cache (`/output/cache/`)
Storage for heavy files to avoid redundant downloads and allow offline execution:
* **`pbf_cache/`**: Regional PBF files (Geofabrik).
* **`ghsl_cache/`**: Global tiles from the Global Human Settlement Layer (population).

---

## Technical Notes
* **Reference System:** Geospatial outputs use geographic coordinates (WGS84), except when local area calculations are required by the pipeline.
* **Reprocessing:** To update the files in this folder, use the `--rerun` flag when executing `main.py`.
* **Privacy and Licensing:** Data in `pbf_cache` is derived from OpenStreetMap and is subject to the [ODbL](https://www.openstreetmap.org/copyright) license.
